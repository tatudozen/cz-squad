# Story 1.2.5: Briefing Capture Form (Astro Component)

**Epic:** 1 - Foundation & Core Module
**Story ID:** 1.2.5
**Status:** Draft
**Priority:** P0 (Blocking)
**Story Points:** 8

## Story Statement

**Como** cliente,
**quero** preencher um formulário intuitivo e validado com meus dados de briefing,
**para que** meus dados sejam capturados corretamente antes de serem salvos no banco de dados via API.

## Acceptance Criteria

1. ✅ Formulário Astro renderizado com campos obrigatórios (business_name, segment, target_audience, voice_tone, objectives, differentiators)
2. ✅ Validação de campos em tempo real usando Zod (mesmo schema de 1.3)
3. ✅ Submit do formulário chama POST /briefings com client_id e dados validados
4. ✅ Feedback visual de sucesso (mensagem de confirmação, reset do formulário)
5. ✅ Feedback visual de erro (exibir validação + erro da API)
6. ✅ Suporte a autenticação X-API-Key (header enviado com requisição)
7. ✅ Responsivo em mobile (Tailwind CSS)
8. ✅ Testes E2E cobrindo: submit válido, validação, erro de API

## Scope

### In Scope
- Astro component para formulário de briefing (`BriefingCaptureForm.astro`)
- Validação de campos com Zod (frontend)
- Chamada POST /briefings com client_id (hardcoded ou via URL param)
- Tratamento de respostas (sucesso 200, erro 400/403/404/500)
- Feedback visual com Tailwind CSS
- Estados de loading/submitting
- E2E tests com Playwright
- Documentação do formulário

### Out of Scope
- Multi-step form wizard
- Salvamento de rascunho
- Histórico de formulários
- Analytics ou rastreamento de submissão
- Integração com n8n direto (vai via API apenas)
- Campos dinâmicos/customizáveis
- Autosave/debounce

## Technical Specification

### Form Structure

```typescript
interface BriefingFormData {
  business_name: string;        // Required: nome do negócio
  segment: string;              // Required: healthcare, fintech, ecommerce, education, etc.
  target_audience: string;      // Required: quem é o público-alvo
  voice_tone: string;           // Required: professional, casual, empathetic, urgent
  objectives: string[];         // Required: array de objetivos (increase-leads, boost-sales, etc.)
  differentiators: string;      // Required: o que torna o negócio único
  monthly_budget?: number;      // Optional: orçamento mensal
```

### Validation Schema (Shared with API)

```typescript
// Same as apps/api/src/schemas/briefing-request.ts
const briefingFormSchema = z.object({
  business_name: z.string().min(3).max(100),
  segment: z.enum([
    'healthcare',
    'fintech',
    'ecommerce',
    'education',
    'saas',
    'retail',
    'other'
  ]),
  target_audience: z.string().min(5).max(200),
  voice_tone: z.enum([
    'professional',
    'casual',
    'empathetic',
    'urgent',
    'friendly'
  ]),
  objectives: z.array(
    z.enum([
      'increase-leads',
      'boost-sales',
      'build-awareness',
      'educate-audience',
      'other'
    ])
  ).min(1),
  differentiators: z.string().min(10).max(300),
  monthly_budget: z.number().positive().optional(),
});
```

### API Call

```typescript
// Form submit handler
const handleSubmit = async (formData: BriefingFormData) => {
  const response = await fetch('/briefings', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': import.meta.env.PUBLIC_API_KEY // ou passar via props
    },
    body: JSON.stringify({
      client_id: clientId, // passed as prop or from URL
      ...formData
    })
  });

  if (response.ok) {
    const data = await response.json();
    // Show success message
    // Reset form
    return data; // Return briefing ID
  } else {
    const error = await response.json();
    // Show error message with details
    throw error;
  }
};
```
